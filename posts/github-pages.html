为什么从 GitHub Pages 开始个人博客

个人博客最难的部分往往不是技术，而是开始。GitHub Pages 把部署压到很轻：维护一个仓库，剩下的交给 GitHub。

我选择先从 GitHub Pages 开始，是因为它让博客的成本足够低。没有服务器账单，没有数据库迁移，也不用先搭一整套内容管理系统。只要能写 HTML，博客就能上线。

它适合什么人

如果你希望拥有一个长期可维护的写作空间，又不想一开始就处理服务器、证书、备份和运行环境，静态博客就是一个很稳的起点。

当前博客采用纯静态结构：`index.html` 是首页，`posts/` 存放文章，`styles.css` 管理样式，`script.js` 负责搜索和筛选。

为什么不用复杂框架

一开始最重要的是降低维护摩擦。纯静态页面足够透明，每个文件都能直接打开、修改和发布。等文章数量变多，再升级到 Jekyll、Astro、Hugo 或 Eleventy 也不迟。

接下来怎么维护

新增文章时，在 `posts/` 目录复制一篇现有文章作为模板，再把首页文章列表、归档、RSS 和站点地图各加一条记录。这个流程虽然朴素，但可控。
